# 問題記錄 — 2026-05-22

## 1. 遊戲掉幀
- **現象**：蛇移動時畫面卡頓、掉幀
- **原因**：`progress` 是 Vue reactive `ref`，rAF 循環每秒更新 60 次，觸發所有蛇節點的 `:style` 重新求值與模板比對
- **修復**：改為普通 `let` 變數，rAF 中直接操作 DOM `style.transform`

## 2. 蛇身一格一格、轉彎斷開
- **現象**：方格蛇身轉彎時出現缺口
- **原因**：每個節點是獨立 25×25 方塊，轉彎處僅角落相接
- **修復**：改為 27px 圓形 (`border-radius: 50%`)，相鄰重疊 2px，形成連續管狀

## 3. 第一節身體被蛇頭吃掉長度
- **現象**：第一節身體明顯比後面的短
- **原因**：橋接方式（bridge）以頭部中心為端點，一半藏在蛇頭內
- **修復**：捨棄橋接方案，全部改用圓形

## 4. 雙人模式 WASD 按下去就卡死
- **現象**：按下 WASD 後遊戲停止、蛇不動
- **原因**：`e.code.slice(3)` 取得 `W`/`A`/`S`/`D`，但 `keyToPos` 只認 `UP`/`DOWN`/`LEFT`/`RIGHT`，導致 `pl.dir` 為 `undefined`，蛇頭位置變成 `NaN`
- **修復**：新增 WASD 方向對照表

## 5. 死掉的蛇卡在牆上抖動
- **現象**：蛇撞牆死亡後，頭部在牆邊快速抖動
- **原因**：rAF 的 `updateSegmentPositions` 未過濾 `gameOver` 玩家，方向持續往牆外推，每次 tick 重設 `lastTick` 又彈回
- **修復**：在 rAF 中加入 `if (pl.gameOver) return`

## 6. 雙人模式左右控制相反、標籤錯亂
- **現象**：左邊 board 的 snake、food、controls 對應的玩家不一致，死蛇變成活蛇傀儡、吃不到食物
- **原因**：`players` array 順序與 template 的 `bi === 0 ? 1 : 0` 映射造成 board 顯示與控制錯位，加上 WASD 原本壞掉造成混淆
- **修復**：統一將方向鍵對應右 board（P1）、WASD 對應左 board（P2），修正標籤與分數顯示

## 7. 死蛇變成活蛇替身（puppet 問題）
- **現象**：一隻蛇死後，活蛇的移動會同步顯示在死蛇的 board 上，看起來像死蛇被操控
- **原因**：`updateSegmentPositions` 中 `players.forEach((pl, pi))` 直接對應 `containers[pi]`，但 template 使用 `bi === 0 ? 1 : 0` 反轉了索引，rAF 把活蛇的位置寫到死蛇的 board 上
- **修復**：改為 `containers[1 - pi]` 以匹配 template 的索引映射

## 8. 左右 board 交換（WASD 改左、方向鍵改右）
- **現象**：玩家希望 WASD 控制左邊蛇、方向鍵控制右邊
- **原因**：上一輪修復後反過來了（WASD 在右、方向鍵在左）
- **修復**：將 key handler 的 mapping 互換，並更新標籤顯示

## 9. 單人/自由模式渲染方式與雙人不一致
- **現象**：單人/自由模式的 board 模板寫死 `cells[0]`、`segmentStyles[0]`，與雙人模式的統一迭代結構不同
- **原因**：歷史遺留，三種模式各有不同的 template 實作
- **修復**：統一 `cells` computed 回傳結構（一律 `players.map`），模板全部改用 `v-for="(_, bi) in players"`，rAF mapping 依 mode 決定是否反轉
